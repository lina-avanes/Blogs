Blog 4:Scanning for updates via powershell

In today's blog, I will demonstrate how to check for windows updates via powershell. The commands demonstrated in this blog, will allow users to scan for updates without actually having to install them. Although they will have the option of installing.

Step #1:
Set the execution policy to remote signed. 
Set-ExecutionPolicy RemoteSigned
RemoteSigned requires that all scripts and configuration files downloaded from the Internet are signed by a trusted publisher. The default execution policy for Windows server computers.


Step #2:

The next command is 


Install-PackageProvider NuGet
NuGet is an open source windows provider program that allows you to manage windows machines For . NET (including . NET Core), the Microsoft-supported mechanism for sharing code is NuGet, which defines how packages for . NET are created, hosted, and consumed, and provides the tools for each of those roles.


Step #3:

The third command is: 

Install-Module PSWindowsUpdate

This command allows you to control, detect, and even install windows updates on powershell.


Step #4:

The third command is: 

Get-WindowsUpdate

The Get-WindowsUpdate command supports querying the local computer or remote computers. When run, you can see below that it returns a list of all of the missing updates on whatever computer you target
